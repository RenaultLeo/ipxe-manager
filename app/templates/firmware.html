Configuration serveur DHCP

Pointez votre serveur DHCP vers les fichiers compilés dans {{ tftp_root }} :

Pour les VMs Proxmox/QEMU/VMware, utiliser snponly.efi — il délègue aux drivers réseau de l'EFI (virtio-net, e1000…) contrairement à ipxe.efi qui embarque ses propres drivers (plutôt pour le bare-metal). 
# pfSense → Services > DHCP Server > [interface]
# Coller dans "Additional BOOTP/DHCP Options" tout en bas :

option client-architecture code 93 = unsigned integer 16;

if exists user-class and ( option user-class = "iPXE" ) {
  # Client déjà sous iPXE → menu direct
  filename "http://{{ menu_url.split('//')[1].split('/')[0] }}/menus/menu.ipxe";
} elsif option client-architecture = 00:07 {
  # UEFI x86_64 — VMs : snponly.efi  |  bare-metal : ipxe.efi
  filename "snponly.efi";
} elsif option client-architecture = 00:0b {
  filename "snponly.efi";   # UEFI ARM64
} else {
  filename "undionly.kpxe"; # BIOS / Legacy
}

# Next server (TFTP) : définir aussi dans le champ "Next Server"
# de la section Network Booting → {{ tftp_root.split('/')[-1] | default('192.168.2.6') }}

Flux de boot : DHCP → TFTP (snponly.efi) → iPXE refait DHCP → reçoit directement {{ menu_url }} grâce à la condition user-class = "iPXE".
